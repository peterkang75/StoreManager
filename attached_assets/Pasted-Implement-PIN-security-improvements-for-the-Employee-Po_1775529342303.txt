PART 2 — PIN ATTEMPT RATE LIMITING (5 attempts = lockout)
═══════════════════════════════════════════════════════

Implement in-memory attempt tracking (no DB changes needed).

Add this at the top of server/routes.ts (after imports):

  const pinAttempts = new Map<string, { count: number; lockedUntil: number | null }>();
  const MAX_PIN_ATTEMPTS = 5;
  const LOCKOUT_DURATION_MS = 15 * 60 * 1000; // 15 minutes

Create a helper function:

  function checkPinRateLimit(identifier: string): { allowed: boolean; minutesLeft?: number } {
    const now = Date.now();
    const record = pinAttempts.get(identifier);
    if (!record) return { allowed: true };
    if (record.lockedUntil && now < record.lockedUntil) {
      const minutesLeft = Math.ceil((record.lockedUntil - now) / 60000);
      return { allowed: false, minutesLeft };
    }
    if (record.lockedUntil && now >= record.lockedUntil) {
      pinAttempts.delete(identifier);
      return { allowed: true };
    }
    return { allowed: true };
  }

  function recordPinFailure(identifier: string): void {
    const now = Date.now();
    const record = pinAttempts.get(identifier) ?? { count: 0, lockedUntil: null };
    record.count += 1;
    if (record.count >= MAX_PIN_ATTEMPTS) {
      record.lockedUntil = now + LOCKOUT_DURATION_MS;
    }
    pinAttempts.set(identifier, record);
  }

  function clearPinAttempts(identifier: string): void {
    pinAttempts.delete(identifier);
  }

Apply to all 3 login routes:
  - identifier = req.body.pin (for login-pin route)
  - identifier = req.body.employeeId (for portal/login route)
  - identifier = req.body.pin (for mobile/auth route)

  At the START of each login handler, before PIN check:
    const limitCheck = checkPinRateLimit(identifier);
    if (!limitCheck.allowed) {
      return res.status(429).json({
        error: `Too many failed attempts. Try again in ${limitCheck.minutesLeft} minutes.`
      });
    }

  On PIN FAILURE: call recordPinFailure(identifier)
  On PIN SUCCESS: call clearPinAttempts(identifier)

Frontend (EmployeePortal.tsx) — PinLogin component:
  The error display already handles string errors.
  Just make sure the error message from the API is shown as-is.
  No UI changes needed — the error text will read:
  "Too many failed attempts. Try again in X minutes."
