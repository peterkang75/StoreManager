You are an expert full-stack engineer.  
Create a full-stack web application for a multi-store business to manage hiring, employee details, and basic internal workflows.

This is Phase 1 only.  
Later phases (roster, time tracking, payroll, cash, suppliers) will be added, so structure code cleanly.

Use the following tech stack:

## Tech Stack
- Backend: Node.js + Express
- Database: SQLite
- ORM: optional (use raw SQL or a lightweight ORM)
- Frontend: React (Vite or CRA)

# =========================================================
#   BACKEND REQUIREMENTS (Express API)
# =========================================================

## Stores API
- GET /api/stores
- POST /api/stores
- PUT /api/stores/:id

## Candidate API
- GET /api/candidates
- GET /api/candidates/:id
- POST /api/candidates            → create
- PUT /api/candidates/:id         → update
- POST /api/candidates/:id/hire   → mark hire_decision = 'HIRE'
  - Generate onboarding token:
    - token = secure random string
    - expires_at = now + 14 days
  - Return onboarding URL: `/m/onboarding/<token>`

## Onboarding API
- GET /api/onboarding/:token
  - Validate token: exists, not expired, not used.
  - Return candidate data as default values.

- POST /api/onboarding/:token
  - Validate token.
  - Create employees row with submitted fields.
  - Save optional uploaded files:
    - Selfie
    - Passport cover
    - Signature
  - Insert into employee_documents table.
  - Mark token used.
  - Return created employee.

## Employees API
- GET /api/employees           → with filters: store_id, status, keyword
- GET /api/employees/:id
- PUT /api/employees/:id       → update (rate, bank info, store, status, etc.)

No delete; use status = INACTIVE.
